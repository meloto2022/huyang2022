# 获客文案库 Skill

> 专为智能家居、灯光设计等本地生活服务行业打造的短视频获客文案生成技能

## 简介

这是一个基于 Claude Code 的 Skill，用于快速生成获客型短视频文案。特别针对**智能家居**和**灯光设计**行业，融入**贵阳本地化**元素，让文案更具说服力和亲切感。

### 核心特点

- **30-60秒口播长度** - 字数150-250字，符合短视频节奏
- **爆款文案结构** - 钩子+痛点+解决方案+CTA，符合短视频传播规律
- **贵阳重度本地化** - 具体楼盘、商圈、本地案例，增加可信度
- **10种专业模板** - 覆盖痛点型、对比型、场景型等多种类型

## 适用人群

- 智能家居行业从业者
- 灯光设计师
- 本地生活服务商家
- 短视频内容创作者

## 文案特点

### 标题类型

- 数字+利益点："3个让贵阳业主后悔的智能家居决定"
- 对比反差："同样花5万做灯光，为什么他家像样板间你家像出租屋"
- 本质揭示："智能家居的本质是生活方式"

### 文案结构

```
标题（吸引点击）
├─ 3秒钩子（提问/痛点/数据/故事）
├─ 痛点共鸣（描述用户遇到的问题）
├─ 案例故事（真实案例增加可信度）
├─ 方法/观点（提供解决方案）
├─ 对比验证（前后对比，数据支撑）
└─ 行动引导（评论区扣"方案"、私信等）
```

## 文案示例

### 智能家居文案示例

```
标题：3个让贵阳业主后悔的智能家居决定，装修前必看

你们肯定想象不到我去年遇到一个花果园业主，装修完才想做智能，结果全屋拆墙重装，多花了3万多。

第一个后悔，没预留零线。装修时电工说单控够了，结果现在想换智能开关，全部要重新布线。

第二个后悔，网口不够。花果园户型小，AP只能用无线回程，智能家居设备多的时候经常断线。

第三个后悔，协议选错了。一开始图便宜用了WiFi设备，结果一断网全部瘫痪，现在换成Zigbee又花了一笔。

记住，装修前智能家居规划三要素：网络、点位、协议。宁可多留，不要少留。

装修前免费方案，评论区扣"方案"。

---

字数：208字 | 预计时长：50秒
```

### 灯光设计文案示例

```
标题：同样花5万做灯光，为什么他家像样板间你家像出租屋

昨天去观山湖一个工地验收灯光，两个业主同样的户型，同样的装修预算，灯光效果天差地别。

第一家，一盏大灯照全屋，惨白惨白的。看电视反光，吃饭没氛围，躺在床上刺眼。

第二家，筒灯、灯带、射灯，层次分明。看电视有观影模式，吃饭有情调，躺在床上温馨舒适。

差别在哪里？不在灯具档次，在灯光设计。

三个专业要点：

第一，色温要分区。客厅3000K暖白，餐厅2700K暖光，卧室要更暖。

第二，层次要分明。基础照明、重点照明、氛围照明，三缺一不可。

第三，显色指数要够。Ra>90，食物才好看，肤色才自然。

贵阳展厅在观山湖，来现场体验不同灯光效果，评论区扣"体验"。

---

字数：218字 | 预计时长：55秒
```

## 自定义修改

### 修改本地化数据

如果你想适配其他城市，只需修改以下文件：

1. **local-data/guiyang-locations.md** - 修改为你所在城市的楼盘/商圈
2. **local-data/guiyang-cases.md** - 添加本地真实案例

### 添加新行业模板

在 `templates/` 目录下创建新文件夹，参考现有模板结构即可。

## 注意事项

1. 本技能基于"铁子一人公司"爆款文案结构分析创建
2. 案例需真实，不能夸大宣传
3. 价格要合理，符合市场行情
4. 定期更新本地数据和案例
